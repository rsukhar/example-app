# example-app

Образец приложения Стартап-Бюро на Laravel 11 + Vue3 + InertiaJS.

> Правила в этом файле имеют **приоритет** над секцией `<laravel-boost-guidelines>` ниже.

## Команды

```bash
./workflow/install-local.sh   # первичная установка
./workflow/up.sh              # запуск (Docker + миграции + Vite)
sail down                     # остановка
```

После установки приложение доступно на http://localhost/

## Стек

- **Laravel 11** — PHP-фреймворк
- **Vue3** + **InertiaJS** — фронтенд, монолитная архитектура
- **StartupUI** — библиотека UI-компонентов (`startup-ui`)
- **VueUse** — вспомогательные composables
- **SASS** + CSS-переменные — стили
- **PostgreSQL** — основная БД
- **Redis** — кэш, временные данные, очереди

## Структура фронтенда

```
resources/views/
  pages/      # страницы — рендерятся через Inertia::render('Folder/Page')
  layouts/    # лейауты
  blocks/     # переиспользуемые компоненты
resources/css/
  app.scss
  variables.scss
  mixins.scss
```

## Структура бэкенда

```
app/
  Http/Controllers/          # тонкие контроллеры
  Http/Requests/ModelName/   # Form Request классы сгруппированы по модели
  Models/                    # толстые модели
  Policies/                  # политики авторизации
  Helpers/                   # вспомогательные классы
```

## Авторизация

Используются Policy-классы (`app/Policies/`), вызываются через `Gate::authorize()`:

```php
Gate::authorize('viewAll', User::class);  // статический метод политики
Gate::authorize('update', $user);         // метод политики с экземпляром
```

## Фильтрация

Каждая модель с фильтрацией имеет `scopeFilter()`. В контроллере:

```php
User::query()
    ->filter($request->only(['role', 'q']))
    ->paginate(5);
```

## Кодстайл

- PHP: PSR-12
- Vue: только Composition API (`<script setup>`), без TypeScript
- Без TailwindCSS, без Options API

# Laravel 11

- CRITICAL: ALWAYS use `search-docs` tool for version-specific Laravel documentation and updated code examples.

## Laravel 11 Structure

- In Laravel 11, middleware are no longer registered in `app/Http/Kernel.php`.
- Middleware are configured declaratively in `bootstrap/app.php` using `Application::configure()->withMiddleware()`.
- `bootstrap/app.php` is the file to register middleware, exceptions, and routing files.
- `bootstrap/providers.php` contains application specific service providers.
- No app\Console\Kernel.php - use `bootstrap/app.php` or `routes/console.php` for console configuration.
- Commands auto-register - files in `app/Console/Commands/` are automatically available and do not require manual registration.

## Database

- When modifying a column, the migration must include all of the attributes that were previously defined on the column. Otherwise, they will be dropped and lost.
- Laravel 11 allows limiting eagerly loaded records natively, without external packages: `$query->latest()->limit(10);`.

### Models

- Casts can and likely should be set in a `casts()` method on a model rather than the `$casts` property. Follow existing conventions from other models.
