# Стандарты кодирования

### Именование (весь код)
- Методы и функции начинаются с глагола: `getLeadsCount` ✓, `leadsCount` ✗
- Булевы методы/свойства начинаются с `is` или `has`
- Единственный экземпляр класса именуется по классу: `$project` для `Project`; несколько экземпляров отражают их роль
- Синонимы для одной сущности внутри одного контекста не используются
- Код читается как предложение на естественном языке: `if ($user->isActive())`
- Единообразное именование по всей цепочке: колонка БД → модель → DTO → контроллер → вью
- Аргументы методов — явные, не упаковывать в массивы/DTO без реальной необходимости

### PHP (PSR-12 + Laravel conventions)
- Использовать constructor property promotion
- Обязательные явные объявления возвращаемого типа
- Каждый класс должен иметь docblock-описание
- Комментарии на русском языке
- Валидация — через Form Request классы, не инлайн
- Тонкие контроллеры, толстые модели — логика, связанная с моделью, живёт в модели
- Контроллеры используют Dependency Injection; модель резолвится по route ID сразу в сигнатуре метода
- Контроллеры возвращают данные Inertia с явно именованными полями, никогда `Model::all()` или полные коллекции
- Использовать `Model::query()` вместо фасада `DB::`
- Геттеры/сеттеры только если нужна дополнительная логика при получении/установке значения
- Транзакции БД только при реальном риске нарушения консистентности данных
- Переменные окружения только в конфиг-файлах, доступ через `config()`
- Документировать формат нестандартных атрибутов (например, ассоциативных массивов) в комментариях

### Vue/Фронтенд
- Порядок блоков SFC: `<template>`, `<script>`, `<style>`
- Стили — SASS-вложенность внутри класса родительского блока (без `scoped`)
- Цвета только из переменных — никогда не добавлять новые захардкоженные значения цветов
- BEM-именование: `.s-blockname` (блок), `.s-blockname-element` (элемент), `.layout_ver` (модификатор)
- Атомарные UI-элементы (`.s-button`, `.s-table`, `.s-alert`) — самостоятельные блоки, никогда не элементы другого блока
- Стандартная структура страницы: `#app.s-app > header.s-header(.s-subheader) + .s-main(.s-section) + footer.s-footer`
- Никаких `console.log` и отладочного кода в коммитах
- Никаких закомментированных фрагментов кода
- Удалять неиспользуемые переменные, props, методы
